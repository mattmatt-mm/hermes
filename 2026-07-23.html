Cache locality: the hidden engine of fast code

  • Spatial locality — Accessing contiguous memory keeps data in cache lines and avoids costly trips to main memory.
  • Temporal locality — Recently used data is likely to be reused, so keep hot structures warm and in-cache.
  • Cache lines and false sharing — Understanding 64-byte lines prevents threads from silently invalidating each other's caches.
  • Data-oriented design — Laying out structs for cache-friendly traversal beats clever object-oriented abstractions.
  • Measuring it — Tools like perf and cachegrind turn cache misses into concrete, actionable numbers.
